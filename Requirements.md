#  inDrive Ride Booking System

## Functional Requirements
FR-1: User shall register
FR-2: User shall login
FR-3: User shall request ride
FR-4: User shall enter pickup/drop location
FR-5: User shall offer price
FR-6: Driver shall accept/reject ride
FR-7: User shall track driver
FR-8: User shall cancel ride
FR-9: System shall calculate fare
FR-10: User shall view history

## Non-Functional Requirements
NFR-1: Response time < 2 sec
NFR-2: Support 5000 users
NFR-3: Secure system
NFR-4: Available 24/7
NFR-5: Real-time tracking
